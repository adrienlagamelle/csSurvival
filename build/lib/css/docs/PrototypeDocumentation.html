Prototype Documentation

    Prototype Requirements:

    1. User password authentication.
    2. Creation and editing of their own postings and topics by authenticated users.
    3. User subscription to topics and subsequent notification of new posts.
    4. Creation of discussion groups (selected users) and optional restriction of topics to created groups.

    Use Case:

    Title: Using Prototype

    Primary Actor: Tester

    Scope: All user topics and posts

    Level: User goal

    Stakeholders and Interests

    User: Would like to use the prototype as a testing environment to test the primary features of the webpage and its design and implementation.

    Precondition: The user has loaded the webpage and has signed in to their created account.

    Minimal guarantee: The tester will be able to view any parts of the webpage that are malfunctions or are poorly implemented

    Success guarantee: The tester will be able to consider how future additions or changes can be made to the webpage to aide in designing the final product.

    Main success scenario:

    • The tester loads the webpage and logs into thier account.
    • The tester examines the design and implementation of both current and future features as outlined in the requirements.
    • The tester decides if this is this information reflects the path of the projects current development.
    • The tester can use this information to shape the final product.

    User Story (Testing Subscriptions):

    The user logs into their account and finds a thread that they would like to subscribe to then they select the subscribe option. After the button is pressed the subscribe option will change to an unsubscribe option notifying the user that they have successfully subscribed to a thread on the webpage. The user will then receive notification of new postings on the given thread.
