Use Case

Title: Editing a post

Primary Actor: User

Scope: All user topics and posts

Level: User goal

Stakeholders and Interests

User: Would like to edit a post or comment, that they have posted to the webpage, to become visible to all other users on the webpage.

Precondition: The user has loaded the webpage and has signed in to their created account.

Minimal guarantee: The webpage will notify the user if the new edited post has failed to appear on the webpage.

Success guarantee: The webpage will replace the post that was edited with the new post leaving it in the same location.

Main success scenario:

  • User selects a post or topic, that they have preciously posted to edit.
  • The user selects to edit the post.
  • The user creates a post to replace the previous post and decides to post it to the webpage.
  • The webpage is updated to display the current information replacing the original post.

Extensions:

  • A failure of any sort while uploading the new edited post to the webpage.
    • 1. The system will notify the user of the error in updating the post.

User Story

The use logs into their account and finds something in one of their postings that they would like to change. They select the post then choose to edit the comment. Once the new post has been created the user decides to upload this new post to the webpage. The new post in now visible to all other users of the webpage.
